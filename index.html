What is a framework anyway?

The basic structure underlying a system.

Bootstrap is a front-end framework:

  • HTML
  • CSS
  • JS

Why use Bootstrap?

  • Responsive and mobile-first
  • Pre-built utility classes, base styles, normalization, and more
  • Components for common page elements (nav bars, media objects)
  • jQuery plugins
  • LOTS of documentation, examples, and community support

When is Bootstrap not the best choice?

  • Highly customized styling
  • Single page, minimally-styled site with no need for extras (but you could use just the parts of Bootstrap that are necessary!)

Mobile first

  • It's easier to add things in than to take them out.
  • Progressive enhancement vs. graceful degradation.

What comes with Bootstrap?

Grid

Grid

  • getbootstrap.com/css/#grid-example-basic
  • Rows create horizontal groups of columns
  • Rows must be placed within a .container or .container-fluid
  • Rows can be nested within columns
  • Classes are based on screen size (xs, sm, md, lg)
  • Media queries:
    • xs: less than 768px
    • sm: 768px and up
    • md: 992px and up
    • lg: 1200px and up
  • If not given a .col-* class for a smaller screen size, the column will stretch to full-width (like .col-xs-12 would).

Let's try it all out!

  1. Create a folder name resume on your computer.
  2. Move the /css, /js, and /font folders you downloaded earlier into /resume.
  3. Create an index.html file inside /resume. You can copy and paste the Bootstrap starter file from getbootstrap.com.

Build out your main page elements.

  1. Create a div at the top using the .jumbotron component. Include your name within an <h1>, and an additional <p> with a subtitle if you'd like.
  2. Add a section with a .container inside of it.
  3. Within the .container, create a grid that has 2 columns, one that is 25% of the page width and one that is 75% (Don't forget the wrapper classes used to make grid rows and columns!)
  4. Copy this section until you have 4 of them.

Fill in the sections!

  1. Now we need to fill out our sections! You can include whatever you'd like, but we'll start with:
    • About Me
    • Skills
    • Experience
    • Education
  2. Within the smaller column for each section, add the name of that section in a <h2> tag.
  3. Give each section an appropriate id; this is how we will navigate to sections later when we implement a navbar.

About Me Section

  1. Including an image of yourself sounds like a good idea. Let's add two columns within the about me section, one for text and one for an image.
  2. In the smaller column, you can add an image of yourself. Choose one of the Bootstrap image classes to style your picture. Make sure it's also responsive!
  3. In the larger column, add a few paragraphs about yourself.
  4. Make sure these columns stack one on top of the other on smaller screens.

Experience Section

  1. Let's add some nested columns so we can display dates alongside your relevant experience. Add 2 columns to the Skills section, one that is 1/3 the total width and one that is 2/3 the total width.
  2. Make sure these columns stack on smaller screens (what classes help you do that?)
  3. In the smaller column, add your job title, location, and dates.
  4. In the larger column, add the details of your position as a bulleted list.
